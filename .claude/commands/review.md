# 코드 리뷰

다음 파일/코드를 리뷰해주세요:
$ARGUMENTS

## 리뷰 체크리스트

### 디자인 원칙
- [ ] **SRP (단일 책임 원칙)**: 클래스가 하나의 책임만 가지는가?
- [ ] **OCP (개방-폐쇄 원칙)**: 확장에 열려있고 수정에 닫혀있는가?
- [ ] **LSP (리스코프 치환 원칙)**: 상위 타입을 하위 타입으로 치환 가능한가?
- [ ] **ISP (인터페이스 분리 원칙)**: 인터페이스가 단일 목적인가?
- [ ] **DIP (의존성 역전 원칙)**: 추상화에 의존하는가?
- [ ] **디미터의 법칙**: 기차 참사 패턴(`a.b().c().d()`) 없는가?

### Unity 최적화
- [ ] `GetComponent`가 `Start()`에서 캐싱되는가?
- [ ] `Camera.main`이 캐싱되는가?
- [ ] `Update()`에서 불필요한 할당이 없는가?

### 코드 품질
- [ ] 네이밍 컨벤션 준수 (PascalCase, _camelCase)
- [ ] 함수가 한 가지 일만 하는가?
- [ ] 죽은 코드, 미사용 변수가 없는가?

## 출력 형식
한국어로 리뷰 결과를 작성하고, 개선이 필요한 부분은 구체적인 코드 예시와 함께 제안해주세요.
